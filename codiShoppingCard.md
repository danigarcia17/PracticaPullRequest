## ¿Qué es una Pull Request?

Una Pull Request es una solicitud para fusionar los cambios de una rama con otra (normalmente `main`) y que alguien los revise antes de integrarlos.

### ¿Para qué sirve?

- Permite revisar el código antes de unirlo.
- Facilita comentar y proponer mejoras.
- Deja un historial claro de qué se cambió y por qué.

### Ventajas

- Mejora la calidad del código gracias a las revisiones.
- Favorece el trabajo en equipo y el aprendizaje.
- Evita romper la rama principal con cambios sin revisar.

# ¿Qué es GitLab?

GitLab es una plataforma completa para desarrollar software basada en repositorios Git.  
Permite guardar y gestionar el código fuente de un proyecto y trabajar en equipo de forma organizada.

## ¿Para qué sirve?

- Controlar las versiones del código.
- Colaborar con otros desarrolladores mediante ramas y Merge Requests (Pull Requests).
- Automatizar pruebas y despliegues con CI/CD.
- Gestionar tareas e incidencias del proyecto.

## Relación con las Pull Requests

Una Pull Request es una solicitud para fusionar los cambios de una rama con otra (normalmente `main`) y que alguien los revise antes de integrarlos.  
Ayuda a revisar el código, comentar mejoras y mantener un historial claro de cambios.

## Ventajas

- Todo en una sola herramienta (código, issues, CI/CD).
- Facilita el trabajo en equipo y el flujo DevOps.
- Mejora la calidad del código gracias a revisiones y Pull Requests.
